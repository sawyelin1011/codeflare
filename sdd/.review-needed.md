### OPEN -- Bisync filter contents not REQ-governed (LOW, from spec-reviewer)

PR #360 broadened the rclone bisync filter in `entrypoint.sh` to exclude
`.wrangler/**` and `.config/**` after a production leak of ~7000 stale
files. No REQ in `sdd/` enumerates the bisync filter contents at the
per-entry level - REQ-AGENT-026 AC1 only governs `**/graphify-out/**`,
and the broader "regenerable tool state is excluded, durable user data
is synced" principle currently lives only in code comments and
`documentation/storage-and-sync.md`.

Proposal: add an AC to whichever REQ owns session-lifecycle R2 bisync
(likely `sdd/session-lifecycle.md` or `sdd/storage.md`) enumerating the
categorical exclude principles - regenerable XDG tool state, vendor
credential caches, ephemeral session logs - rather than per-path
inventory. The doc lane already lists the paths; the spec lane should
own the *categories* so future filter changes have something to be
verified against.

Not blocking PR #360. Recommended as a follow-up.
